Inclusive

Define the operation method (Inclusive|Exclusive). When you choose exclusive mode (right button), the addon does NOT watch any URL or domain, unless you specify a blacklist rule(s) in table below to block a desired URL or domain. On the other hand, in Inclusive mode (left button), the addon monitors all websites and URLs, unless you specify a whitelist rule below. Default mode is Inclusive.

Check this box to print behind-the-scene info to the browser's console (useful for debugging your rules). Adding rules: write (whitelist or blacklist) rule(s) for URLs with Regular Expressions (RegExp). Any URL that matches these rule(s) will be whitelisted or blacklisted. Please note, place "|" (or operator) between each rule and add as many rules as you need in a single line. For example the following rules will whitelist all URLs in the format of ".test.com/?keyword" or "sample-url.com": sample\-url\.com|\.test\.com\/\?keyword. Note: when using these rules to block a content(s), it is better to DISABLE Content-Security-Policy method above and only use Web-Request-Observer approach. This is because Content-Security-Policy blocks items in group and is stricter than Web-Request-Observer; for example if you select Image category to block with this method, it will block ALL inline and external images within a page, therefore it is not possible to block a single URL or image content with this method. But, Web-Request-Observer can block a single item or URL. Therefore, in order to use blacklist rules, please select Web-Request-Observer method only. Visit the addon's support page to get more info about different blocking options.


Regular expressions (RegExp): a regular expression (regex or regexp for short) is a special text string for describing a search pattern. You can think of regular expressions as wild-cards on steroids. It is similar to wild-card notations such as *.txt to find all text files in a file manager. Please visit this Wikipedia page to get more info about regular expressions.

ALL: this icon represents all URLs. In other words, the addon will monitor and block ALL URLs for the selected type (Script, Style, Image, Object, Media). You can select desired HTML type from toolbar-panel UI. Please note: this selection does NOT affect whitelist or blacklist RegExp rules.

THIRD-PARTY: this icon represents only third-party URLs. It will only monitor and block Third-Party URLs within a page for the selected HTML type. Please note: this selection does NOT affect whitelist or blacklist RegExp rules.

TOP + FRAME: this icon represents URLs from Top and all iFrames within the page (TOP + FRAME)). Once this icon is active, the entire URLs for a website are monitored. Please note: this selection does NOT affect whitelist or blacklist RegExp rules.

FRAME: this icon represents URLs from only iFrames within the page. Once this icon is active, only URLs from FRAMEs within a website are monitored. Please note: this selection does NOT affect whitelist or blacklist RegExp rules.
